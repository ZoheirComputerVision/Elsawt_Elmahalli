## بروتوكول النشر (مطلوب في كل جلسة)
### Phase 1 — تطوير محلي
- نفذ التغييرات محلياً فقط
- اختبر محلياً (تشغيل الاختبارات: `node tests/*.test.js`)
- لا تقم بـ commit أو push حتى تؤكد المعاينة

### Phase 2 — معاينة
- أبلغ المستخدم بأن التحديثات محلية وجاهزة للمعاينة
- انتظر تأكيد المستخدم قبل النشر

### Phase 3 — نشر
- commit → push → Render auto-deploy
- فقط بعد موافقة المستخدم

## تنسيق الرسائل
- Commit messages: بالعربية، تصف التعديل بدقة
- أبداً لا تستخدم `-m` بدون وصف واضح
