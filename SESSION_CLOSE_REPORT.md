## 🧭 3. خطة الجلسة القادمة

### أول 3 مهام للجلسة القادمة
1. **Commit + Push:** نشر جميع تعديلات الجلسات 4+5 (إصلاح الجوال + ميزة فيسبوك) إلى GitHub → HostingGuru.io
2. **إعداد Facebook Token حقيقي:** الحصول على `fb_page_access_token` من Facebook Developer Portal (صلاحية `pages_manage_posts`) وتعيينه عبر API `POST /admin/settings`
3. **إضافة واجهة مستخدم إدارية للفيسبوك:** إضافة أزرار "مشاركة على فيسبوك" و "تعليق" في صفحة مراجعة المحتوى + عرض سجل الردود

### قرارات تقنية تم اتخاذها (يجب ألا تتغير)
- القائمة المنسدلة توضع **بعد "الرئيسية" وقبل "الأخبار"** مباشرة
- الروابط الخارجية تفتح في نافذة جديدة (`target="_blank"`) مع `rel="noopener noreferrer"`
- أسلوب CSS يتبع نمط الموجود (متغيرات `--accent`, `--border`, `--radius`, إلخ)
- إيقونات: 🌐 للقائمة الرئيسية، 👤 لفضاء الأولياء، 👨‍🏫 لفضاء الأساتذة، 🏆 للمسابقات، 📚 للتعليم عن بعد، 📝 للتسجيل في إمتحان
- أقسام القائمة: `.nav-dropdown-section` (عنوان غير قابل للنقر)، `.nav-dropdown-divider` (فاصل)
- النشر يتم عبر: **Git commit → Git push → GitHub → HostingGuru.io (deploy تلقائي)**
- القائمة المنسدلة تدعم click/toggle للشاشات اللمسية (مع احتفاظ hover للماوس)
- على الشاشات ≤600px، القائمة تأخذ عرض كامل الشاشة (هوامش 10px) لعدم التجاوز
- الضغط خارج القائمة أو Escape يغلقها
- **جديد (جلسة 4):** استخدام `position: fixed` للقائمة على الجوال مع حساب `top` ديناميكي
- **جديد (جلسة 4):** إضافة `touch-action: manipulation` لمنع تأخير 300ms على اللمس
- **جديد (جلسة 5):** الرد على فيسبوك يعمل في **Simulation Mode** تلقائياً عندما يكون `fb_page_access_token = 'demo'` أو فارغاً (القيمة الافتراضية)
- **جديد (جلسة 5):** مشاركة المقال (share) هي السلوك الافتراضي في `autoReplyAfterPublish()`، والتعليق (comment) متاح يدوياً عبر API

## 💾 4. سياق المشروع (Project Context)

### ملخص المشروع
نظام "الجريدة المدرسية الذكية" هو منصة آلية بالذكاء الاصطناعي لثانوية المجاهد خليل محمد (عين كرمس - تيارت، الجزائر). يقوم بجمع المحتوى من Facebook ووزارة التربية الوطنية، تحليله وتصنيفه عبر AI، ثم نشره تلقائياً مع مراجعة بشرية اختيارية. يحتوي على واجهة عامة، أرشيف زمني، ولوحة تحكم إدارية كاملة. **جديد في الجلسة 5:** يمكن للنظام الآن مشاركة المقالات المنشورة على صفحة الثانوية في فيسبوك تلقائياً أو يدوياً.

### الـ Stack التقني
| الطبقة | التقنية |
|--------|---------|
| **الخادم** | Node.js + Express |
| **قاعدة البيانات** | JSON files (`data/*.json`) — 10 جداول: sources, raw_data, processed_content, media, archive, ai_decision_log, admin_actions, settings, views, fb_replies |
| **الواجهة** | HTML5 + CSS3 (Vanilla) + JavaScript (Vanilla) |
| **الأمان** | Helmet.js + CORS |
| **الجدولة** | node-cron (جمع وتحليل كل 30 دقيقة) |
| **التكامل الخارجي** | Facebook Graph API v18.0 (محاكاة حالياً) |
| **النشر** | GitHub → HostingGuru.io (deploy تلقائي) |

### ملاحظات وقيود
- الموقع بالكامل **RTL بالعربية**
- **لا توجد** واجهة MVC (React/Vue/Angular) — كل شيء Vanilla JS
- قاعدة البيانات **JSON files** — غير مناسبة للتوسّع الأفقي (ليست SQLite كما ورد سابقاً؛ لا يوجد `database.sqlite` حقيقي، هو JsonDB)
- الصور تُخزّن كـ **Base64** في قاعدة البيانات (لا يوجد نظام ملفات حقيقي)
- بيئة التطوير: **Windows** (المسارات بـ `path.join`)
- بيئة الإنتاج: **Linux (HostingGuru.io)** — CRLF → LF تلقائي
- `main.js` يتم تحميله في جميع صفحات الموقع عبر `<script src="/js/main.js">`
- القائمة المنسدلة تعمل بـ CSS hover + JS click toggle معاً (متوافقة مع جميع الأجهزة)
- **جديد:** `fb_reply.js` يستخدم `config.AI.FB_API_VERSION` (18.0) و `config.AI.FACEBOOK_PAGE` لبناء روابط Graph API
- **جديد:** الرد على فيسبوك يتم في الخلفية (fire-and-forget) بعد النشر — لا يمنع عملية النشر إذا فشل
